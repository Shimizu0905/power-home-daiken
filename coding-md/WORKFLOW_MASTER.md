# Figma MCP コーディングワークフロー（全体）

## 概要

Figma MCP を使用した Web サイトのコーディング工程をまとめたマスターファイルです。各工程の詳細は個別の md ファイルを参照してください。

## 前提条件

- Figma MCP が接続されていること
- Figma で対象のデザインが選択されていること
- プロジェクトのセットアップが完了していること（Vite、SCSS 環境など）

## 工程一覧

### 0. MCPログの保存（必須）

**実際の操作に入る前に、必ずMCPログを保存します。**

- Figma MCP で取得したデザイン情報（フォント、カラー、レイアウト、テキスト内容など）は、必ず `coding-md/mcp-log/` ディレクトリに保存してください
- 保存ルール: `[coding-md/mcp-log/README.md](./mcp-log/README.md)` を参照
- これにより、後続の品質チェックやコードとの整合性確認時に、MCP に再接続することなく参照できます

**重要**: 画像取得の指示がない限り、画像は取得しません。デザイン情報のみを取得して保存してください。

### 1. フォント・カラー情報の取得

**[工程 1: フォント・カラー変数の準備](./01-font-color-setup/WORKFLOW_01_FONT_COLOR_SETUP.md)**

- Figma からフォントとカラー情報を取得
- MCP 取得値は `coding-md/mcp-log/` に保存
- Google Fonts CDN の読み込み
- SCSS 変数の登録

### 2. 画像のエクスポート（指示があるときのみ）

**[工程 2: 画像のダウンロードと命名](./02-image-export/WORKFLOW_02_IMAGE_RENAMING.md)**

- **注意**: 画像取得の指示がある場合のみ実施します
- Figma から画像をダウンロード
- `src/public/assets/img/` に保存
- 命名規則に従ってリネーム
- ビルドツールが自動で最適化・出力を行う

### 3. コーディング

**[工程 3: コーディング](./03-coding/WORKFLOW_03_INITIAL_CODING.md)**

- Figma からデザイン構造を確認
- MCP 取得値は `coding-md/mcp-log/` に保存（画像取得の指示がない限り、画像は取得しない）
- **セクションごとに** HTML → SCSS の順でコーディング
- 各セクションで必要な部分だけを取得し、コンテキストを適切なサイズに保つ
- 命名済みの画像パスを使用

### 4. スクリーンショット比較

**[工程 4: スクリーンショット比較と調整](./04-screenshot-comparison/WORKFLOW_06_SCREENSHOT_COMPARISON.md)**

- Playwright で実サイトのスクリーンショットを取得
- Figma MCP でスクリーンショットを取得
- 比較して差異を修正
- 2 回繰り返して精度を上げる

### 5. 品質チェック

**[工程 5: 品質チェック（エディタ内で完結）](./05-quality-check/WORKFLOW_05_QUALITY_CHECK.md)**

- タイポグラフィ、色、画像、JS/アニメーション、フォーム、meta/head、HTML/リンクのコード上の確認
- MCPログ（`coding-md/mcp-log/{page}/design-context.md`）を参照して、実装コードとの整合性を確認

## 工程の順序

```
工程0（MCPログの保存：必須）
    ↓
工程1（フォント・カラー情報の取得）
    ↓
工程2（画像のエクスポート：指示があるときのみ）
    ↓
工程3（コーディング：セクションごとにHTML→SCSS）
    ↓
工程4（スクリーンショット比較と調整 × 2回）
    ↓
工程5（品質チェック：エディタ内で完結）


## 重要事項

### 画像取得について

**画像取得の指示がない限り、画像は取得しません。**

- `get_design_context` を使用する際も、デフォルトでは画像は取得せず、デザイン情報（フォント、カラー、レイアウト、テキスト内容など）のみを取得してください
- 新しい画像が必要な場合のみ、明示的に「○○の画像を取得してください」と指示してください
- 詳細: `[coding-md/mcp-log/README.md](./mcp-log/README.md)` の「画像取得のベストプラクティス」を参照

### MCPログの保存

**すべてのMCP取得値は必ず保存してください。**

- Figma MCP で取得したデザイン情報は、必ず `coding-md/mcp-log/` ディレクトリに保存してください
- これにより、後続の品質チェックやコードとの整合性確認時に、MCP に再接続することなく参照できます
- 保存ルール: `[coding-md/mcp-log/README.md](./mcp-log/README.md)` を参照

## 注意事項

- 各工程は順番に実行すること
- 前の工程が完了してから次の工程に進むこと
- 各工程で問題があれば必ず修正すること
- 画像取得の指示がない限り、画像は取得しないこと

## スクリーンショット取得の自動化

工程 4（スクリーンショット比較と調整）では、スクリーンショット取得の自動化が可能です：

- **Playwright**: `coding-md/screenshot.js` スクリプトで自動取得可能
- **MCP 経由**: Figma からは自動で取得可能

詳細は各工程の md ファイルを参照してください。
